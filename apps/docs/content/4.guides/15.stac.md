Point tileserver-rs at any [SpatioTemporal Asset Catalog (STAC)](https://stacspec.org/) API and serve the underlying Cloud-Optimized GeoTIFFs (COGs) as raster tiles — no ingest, no pre-processing, no local copy.

## Three modes

| Mode | Trigger | Use when |
|------|---------|----------|
| **Static discovery** | `dynamic = false` (default) | One-shot visualisation of the most recent asset. Ingested once on startup. |
| **Dynamic per-tile discovery** | `dynamic = true`, `max_items = 1` | Always-fresh per-tile lookups; tiles come from whichever single asset intersects. |
| **Multi-asset mosaic** | `dynamic = true`, `max_items >= 2` | Composite several overlapping items (cloud-free mosaic, time-stack, multi-sensor). |

Enable the feature at build time:

```bash
cargo build --release --features stac
```

## Static discovery

Point at a STAC API collection; tileserver-rs performs a single `/search` on startup, merges the discovered bounding boxes, and serves the first COG asset.

```toml
[[sources]]
id = "sentinel2"
type = "stac"
path = "https://earth-search.aws.element84.com/v1"
collection = "sentinel-2-l2a"
asset_role = "visual"        # or "thumbnail" / a specific asset key
dynamic = false
max_items = 25               # cap the catalog scan
```

Served at `GET /data/sentinel2/{z}/{x}/{y}.png`.

## Dynamic per-tile discovery

Flip `dynamic = true` and every incoming tile triggers a fresh STAC `/search` with the tile's WGS-84 bounding box:

```toml
[[sources]]
id = "sentinel2-dynamic"
type = "stac"
path = "https://earth-search.aws.element84.com/v1"
collection = "sentinel-2-l2a"
asset_role = "visual"
dynamic = true
max_items = 1                # single best asset per tile
```

Tiles always reflect the current catalog state — useful for dashboards that must show the latest imagery without restarts.

**Cost:** one STAC API call + one COG range-read per unique tile. Pair with a global cache (see `[cache]` config) to amortise hot-path cost.

## Multi-asset mosaic compositing

Bump `max_items` to fetch multiple candidate items and composite them in-memory:

```toml
[[sources]]
id = "sentinel2-mosaic"
type = "stac"
path = "https://earth-search.aws.element84.com/v1"
collection = "sentinel-2-l2a"
asset_role = "visual"
dynamic = true
max_items = 5                # top 5 candidates, composited
```

**Semantics:**
- STAC `/search` is asked for up to `max_items` intersecting items; results come back ranked (newest first, least cloud first — however the API sorts).
- Every candidate COG is rendered to an RGBA tile.
- The **lowest-priority** asset is the canvas; each higher-priority asset overlays on top via straight-alpha `image::imageops::overlay`.
- Top-priority opaque pixels win in overlapping regions; transparent gaps fall through to lower-priority layers.
- Any single failed COG is logged and skipped — one bad asset does not blank the tile.

The result is a seamless cloud-free-ish mosaic without any preprocessing step.

## Element84 Earth Search quick example

A fully working config for global Sentinel-2 L2A imagery via Element84's public API:

```toml
[[sources]]
id = "sentinel2"
type = "stac"
path = "https://earth-search.aws.element84.com/v1"
collection = "sentinel-2-l2a"
asset_role = "visual"
dynamic = true
max_items = 5

[[styles]]
id = "sentinel2"
path = "./data/styles/sentinel2/style.json"
name = "Sentinel-2 L2A"
```

Style JSON references the raster source:

```json
{
  "version": 8,
  "sources": {
    "sentinel2": { "type": "raster", "url": "/data/sentinel2.json", "tileSize": 256 }
  },
  "layers": [
    { "id": "bg", "type": "background", "paint": { "background-color": "#000" } },
    { "id": "sentinel2", "type": "raster", "source": "sentinel2" }
  ]
}
```

## Supported STAC catalogs

Any STAC API that implements the `/search` endpoint (STAC API v1.0+) with a `POST` body accepting `collections`, `bbox`, and `limit`:

- **Element84 Earth Search** — `https://earth-search.aws.element84.com/v1`
- **Microsoft Planetary Computer** — `https://planetarycomputer.microsoft.com/api/stac/v1`
- **USGS Landsat STAC** — `https://landsatlook.usgs.gov/stac-server`
- **NASA CMR STAC** — `https://cmr.earthdata.nasa.gov/stac/…`
- Self-hosted [stac-fastapi](https://github.com/stac-utils/stac-fastapi) / [stac-server](https://github.com/stac-utils/stac-server)

## Asset selection

tileserver-rs picks one COG asset per item using this priority order:

1. Asset with a `roles` array containing the configured `asset_role` value.
2. Asset keyed by the configured `asset_role` string.
3. Asset whose `type` is `image/tiff; application=geotiff; profile=cloud-optimized` or starts with `image/tiff`.

## 3D bounding box handling

STAC items may carry 6-element bounding boxes `[west, south, min_elev, east, north, max_elev]`. tileserver-rs correctly extracts the 2D footprint (`[0, 1, 3, 4]`) so elevation metadata never displaces tiles.
